## Step 2: Firestore Security and Indexes

1. **Firestore rules (Firebase Console)**
   - In Firebase Console → Firestore Database → Rules, use rules that:
     - Allow read/write only when the user is authenticated.
     - Optionally restrict `listings` so that `update`/`delete` are allowed only when `request.auth.uid == resource.data.createdBy`.
   - Example (adjust to your exact rules):
     - `users`: authenticated users can read/write their own document (`doc(userId)`).
     - `listings`: authenticated read; create allowed; update/delete only if `resource.data.createdBy == request.auth.uid`.

2. **Composite index for “My Listings”**
   - The app queries: `listings` where `createdBy == uid` order by `timestamp` desc.
   - A composite index is required. You have two options:
     - **Option A**: Deploy the included indexes:
       ```bash
       firebase deploy --only firestore:indexes
       ```
       (Ensure `firebase.json` includes Firestore and that `firestore.indexes.json` is in the project root; the repo includes an index for `createdBy` + `timestamp`.)
     - **Option B**: Run the app, trigger “My Listings,” and when Firestore throws an error, open the link in the error message to create the index in the Firebase Console.

## Step 4: Verify Each Requirement

### 1. Authentication
- **Sign up**: Create a new account from the Sign Up screen.
- **Email verification**: You cannot use the app until the email is verified (Verify Email screen).
- **Log in / Log out**: Log out from Settings → Sign Out; then log in again with the same account.
- **User profile in Firestore**: After sign-up, a document is created in `users/{uid}` (see `AuthService.signUp`). Listings store `createdBy` (user UID).

### 2. Location Listings (CRUD)
- **Create**: From Directory (Home), tap the **+** (yellow) button, or go to **My Listings** → **Add**. Fill the form and submit.
- **Read**: Directory shows all listings; **My Listings** shows only the current user’s listings (via `createdBy`).
- **Update**: In **My Listings**, tap the **edit** icon on a listing; change fields and save.
- **Delete**: In **My Listings**, tap the **delete** icon; confirm in the dialog.
- All of this goes through `ListingsProvider` and `FirestoreService` (no direct Firestore in UI).

### 3. Directory Search and Filtering
- **Search**: On the Directory screen, type in “Search for a service.” Results filter by name/address in real time (`ListingsProvider.setSearch` + `filtered`).
- **Category**: Use the horizontal chips (Cafés, Pharmacies, etc.). Selecting a category filters the list (`ListingsProvider.setCategory`). Data is still driven by Firestore streams; filtering is in memory from that data.

### 4. Detail Page and Map
- From Directory (or Map), open a listing.
- **Detail page**: Name, category, rating, address, contact, description.
- **Map**: Embedded map with a marker for the listing’s coordinates.
- **Turn-by-turn**: Tap **“Get Directions”** to open Google Maps (or the system map app) with directions to that location (`url_launcher` with `google.com/maps/dir/?api=1&destination=...`).

### 5. State Management
- **Provider** is used: `AuthProvider`, `ListingsProvider` in `main.dart`.
- Firestore is only used inside `FirestoreService`. The UI uses only the providers (e.g. `context.watch<ListingsProvider>()`, `listings.filtered`, `listings.myListings`). Loading and error states are exposed by the providers.

### 6. Navigation
- **BottomNavigationBar** (in `HomeScreen`) includes:
  - **Home** → Directory (browse listings).
  - **Bookmarks** → Saved listings (optional; beyond the minimum).
  - **My Listings** → Current user’s listings (CRUD).
  - **Map** → Map view with all listing markers; tap marker info to open detail.
  - **Settings** → Profile and notification toggle.

### 7. Settings
- **Profile**: Displays the authenticated user’s name and email (from Firestore `users/{uid}` and auth).
- **Location-based notifications**: The **“Location Notifications”** toggle enables/disables a local simulation of location-based notifications (state only; assignment allows local simulation). Other toggles (e.g. Nearby Alerts, Review Notifications) are optional extras.
